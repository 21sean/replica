How Replica works

Everything is local. The Agent is whatever Ollama model you pick. Projects are plain folders in replica/projects/ on your disk.

Building. Describe what you want on the Home screen. The Agent plans, then writes real files, and you will see each one land with a check mark. The preview refreshes automatically.

Iterating. Keep chatting in the project: make the header sticky, add dark mode, fix that bug. The Agent sees your current files each turn, including manual edits.

Code tab. Syntax-highlighted editor over the project files. Ctrl+S saves, and files stream into view live while the Agent writes them. Your edits become part of the Agent's context.

Console tab. Run node file.js or python file.py inside the project folder, with a 60 second limit.

Server apps. Press Run in the toolbar to start a dev server. It gets a free port in PORT, its logs stream to the Console tab, and the preview proxies to it while it runs.

Models. Bigger models build better apps. Thinking models show their reasoning, and you can collapse it with a click. The first response after idle can take a while as the model loads into memory.
